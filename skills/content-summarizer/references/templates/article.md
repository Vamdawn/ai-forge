## Callout 使用指南

| Callout | 用途 |
|:--------|:-----|
| `[!abstract]` | TL;DR 摘要 — 始终放在标题正下方 |
| `[!quote]` | 文章自身的标语或关键引用 |
| `[!important]` | 读者不可错过的关键洞见 |
| `[!tip]` | 实践建议或最佳实践 |
| `[!warning]` | 局限性、偏见、常见陷阱或反面观点 |
| `[!note]` | 补充背景或上下文 |

## 适配规则

- 根据文章深度调整 section 数量（短文章 = 更少的 section）
- 文章无显著外部链接时省略"推荐资源"section
- 文章涉及工具/方案对比时添加对比表格
- 使用 `==highlight==` 标记最关键的 2-3 个短语（谨慎使用）
- 使用 `[[wikilinks]]` 链接到 vault 中可能已有的概念笔记

### 子类型适配

| 文章类型 | 核心观点聚焦方向 | 特殊 section |
|:---------|:---------------|:------------|
| `tutorial` | 步骤序列；保留关键代码和命令 | 添加"前置条件"和"步骤"子节；保留代码块 |
| `opinion` | 论证链：前提 → 推理 → 结论 | 加强"批判性注记"；记录作者核心假设 |
| `research` | 方法论、关键发现、数据亮点 | 添加"方法论"和"局限性"子节 |
| `news` | 5W1H；事实时间线 | 保持简洁；强调时间线和影响 |
| `comparison` | 比较维度和最终结论 | 必须包含对比表格；标出胜者/取舍 |

### 多媒体处理

- **关键图表/架构图**：截图保存到 vault 附件目录，用 `![[filename.png]]` 嵌入，添加文字说明
- **代码块**：保留核心代码（关键逻辑、核心配置）；样板代码用描述替代
- **数据可视化**：尽量提取底层数据为 Markdown 表格；否则保存为图片
